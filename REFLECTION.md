# 反思报告

这次课程项目我把主题确定为“上海迪士尼乐园运营客流预测与商业洞察分析”。一开始我只想做一个客流预测模型，但对照评分标准后，我意识到老师更看重“面向明确用户的小型数据产品”，而不只是几张图或一段模型代码。因此我把项目重新包装成 business 场景下的分析产品，让用户不仅能看到预测人数，还能直接获得运营关注点和游客建议。

在 Python 实现层面，我把流程拆成了数据生成与整理、探索性分析、特征工程、模型训练和前端展示几个部分。因为真实的上海迪士尼逐日客流数据难以公开获得，我采用了模拟数据方案，但尽量让数据与节假日、学校假期、季节、天气和特殊活动等现实因素建立联系。这样做虽然不能替代真实经营数据，但可以完整展示 Python 在数据获取、清洗、转换、分析和输出上的链条。

从分析结果来看，这个项目最有价值的地方并不是“预测出一个数字”，而是把客流变化转成可被理解和使用的判断。例如节假日和学校假期会显著拉高客流，春末和夏季更容易出现高压区间，这些发现都能进一步转化为排班、动线管理和游客建议。相比只给出单日预测值，日期区间分析更接近真实运营决策。

这个项目也暴露出一些局限。最核心的问题是数据仍然是模拟数据，因此模型精度和规律解释只能用于课程展示，不能直接代表真实园区情况。其次，当前还没有接入实时天气、门票价格、社交媒体热度和实际活动排期，因此对突发事件和短期波动的解释能力有限。产品目前也主要是本地运行版本。

在 AI 使用方面，我使用了 AI 工具辅助完成界面文案优化、README 结构整理、调试思路和摘要接口联调。但项目的问题定义、数据流程设计、特征构造、模型比较、分析判断和最终结论都由我自己决定，并结合 Python 输出核查。我也认识到，AI 可以帮助提升效率，但不能替代对数据来源、分析逻辑和结果含义的理解。这次项目最大的收获，是学会了如何把“分析过程”做成一个面向用户的产品表达。
