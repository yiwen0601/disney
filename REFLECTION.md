# 反思报告

本项目围绕“上海迪士尼乐园运营客流预测与商业洞察分析”展开，核心问题不是单纯预测某一天的人数，而是把预测结果转化为可以支持运营决策与出行判断的产品输出。目标用户主要有两类：一类是面向园区运营、排班和资源调度的 business 使用者，另一类是希望比较不同日期区间、降低拥挤风险的普通游客。基于这一定位，我把项目设计成“预测 + 解释 + 建议”的交互式工具，而不是只提交模型结果。

在数据集选择上，最大的现实限制是上海迪士尼并没有公开的逐日官方客流数据，因此我采用了“公开事实约束下的模拟数据”方案。数据时间范围为 2016 年至 2025 年，生成逻辑综合参考了节假日安排、上海中小学假期、季节变化、天气条件和主题活动节奏，同时接入高德天气 API 补充近未来上海天气预报。选择这类数据的原因，是它虽然不能替代真实经营数据库，但能够完整支撑数据清洗、转换、分析、建模与产品表达这一整条 Python 工作流。

在方法与工具上，我主要使用了 Python 生态中的 Pandas、NumPy、Matplotlib、Seaborn、scikit-learn、Flask 和 Joblib。具体流程包括：构造日期与业务字段、检查缺失值和异常值、完成节假日和天气等特征工程、建立滞后与滚动统计特征、进行探索性数据分析，并比较线性回归、Ridge、Lasso、随机森林等模型的表现，最终将较稳定的模型封装到网页应用中，让用户通过日历选择区间并获取可读性更高的分析结果。

从核心洞察来看，节假日和学校假期是影响客流的关键因素，夏季和春末更容易形成高峰区间，天气变化则会对短期波动和建议生成产生辅助影响。项目的价值不在于给出一个“绝对准确”的数字，而在于识别高峰窗口、解释驱动因素，并进一步转化为运营建议和游客建议。

当然，项目仍有明显局限。首先，数据本质上是模拟数据，因此模型可靠性更适合课程展示和方法说明，不适合直接用于真实经营决策。其次，当前尚未系统接入真实历史天气、门票价格、社交媒体热度和实际活动排期，对突发事件的解释能力有限。未来如果能接入更真实的数据源，并加入持续更新机制和线上部署，项目的可信度与可用性都会明显提升。

在个人贡献与学习收获方面，选题定位、数据流程设计、特征构造、模型比较、前端呈现和结果解释都由我主导完成，AI 工具主要用于文案润色、接口调试和效率辅助。这次项目让我更清楚地理解到，数据分析课程的关键不只是“会写代码”，而是能否把问题定义清楚、把数据限制讲明白，并把分析结果转化成面向用户的、有实际使用感的小型数据产品。
